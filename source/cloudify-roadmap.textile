h1. *Cloudify Roadmap | Version 3.2*

h2. Target Date: Q2 2015

h1. <i class="icon-star"></i> *Features*

h2. <i class="icon-star-empty"></i> *Manager data persistency to external volume*

The Cloudify manager state will be persisted to a file system on an external volume in order to ensure more robust data persistency.


h2. <i class="icon-star-empty"></i> *Manager recovery from external volume*

This will enable recovery from crashes, restarts, and open the door for automated upgrades.

h2. <i class="icon-star-empty"></i> *IPv6 support*

IPv6 is much required in the Telecom vertical, and this will further enable NFV functionality.

h2. <i class="icon-star-empty"></i> *Autoscale*

Moving forward from the APIs introduced in 3.1 to policies and workflows, with some additions to the manager core.

h2. <i class="icon-star-empty"></i> *vCloud Support*

3.2 will introduce support for the creation of virtual machines, network settings, and storage over vCloud.

h2. <i class="icon-star-empty"></i> *SoftLayer Support*

 In addition, 3.2 will also introduce support for IBM's SoftLayer IaaS, enabling orchestration from the infrastructure through the application level.

h2. <i class="icon-star-empty"></i> *Docker Plugin*

The GA of the much-discussed and needed Docker plugin will be available in this version.

h2. <i class="icon-star-empty"></i> *Security*

3.2 will bring with it enhanced security features including REST API authentication vs. LDAP and encryption between the CLI and manager.

h2. <i class="icon-star-empty"></i> *GUI*

This version will bring with it some GUI enhancements including an improved view for logs, as well as some UX improvements, so stay tuned.


h2. <i class="icon-star-empty"></i> *Manager distribution into Docker containers per component*



h2. <i class="icon-asterisk"></i> *<em>Premium Feature</em> - Blueprint graphic composer (MVP release)*


<hr/>


h1. *Looking Forward - Future Thoughts*

We have a lot in store in the planning for after the next version as well - so be sure to comment, reach out and provide your input, so we know that we're on the right track.

h2. <i class="icon-star-empty"></i> *TOSCA Enhancements - Support for requirements and capabilities*

 This is a TOSCA feature for validations and auto-wiring of components that serve as an implicit alternative to the existing explicit relationships.

h2. <i class="icon-star-empty"></i> *Support for deployment time governance & SLA policies*

Such as: placement (anti-affinity, affinity, etc), white list / black list of libraries and SW, and more.

h2. <i class="icon-star-empty"></i> *Highly available Cloudify manager with hot failover and leader election*
