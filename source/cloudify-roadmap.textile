h1. *Cloudify Roadmap*

<div class="bs-callout bs-callout-warning"><strong><p>Note</p></strong>
    <p>Note: Cloudify aims for a minor release with new features, bug fixes, security enhancements, and more every 4-6 months.<br>
	A service release with bug and security fixes is also intended for delivery every 1-2 months.</p>
</div>

<p>Below you will find our roadmap for new features and improvements to Cloudify, including notes on those currently being worked on.</p>

<p><b></b>

  <h2>Cloudify Manager Enhancements</h2>

  <ul>
    <li>
      <p>Cloudify Manager upgrade process improvements <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Cloudify Manager high availability </p>
    </li>

    <li>
      <p>Dynamic Manager settings configuration</p>
    </li>

    <li>
      <p>Encrypted data store for credentials to be used by application blueprints </p>
    </li>

    <li>
      <p>Improved Manager bootstrap process <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Pre-bootstrap verification command</p>
    </li>

    <li>
      <p>Built-in authentication user store</p>
    </li>

    <li>
      <p>Application deployment states </p>
    </li>

    <li></li>
  </ul>

  <h2>Security</h2>

  <ul>
    <li>
      <p>Requirement for all communications to travel via SSL <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Access control lists for resources managed by Cloudify</p>
    </li>

    <li>
      <p>Resource multi-tenancy </p>
    </li>

    <li>
      <p>Use Letâ€™s Encrypt SSL certificates </p>
    </li>
  </ul>

  <h2>App Modeling and DSL Improvements</h2>

  <ul>
    <li>
      <p>Ability to model application in separate blueprints and simply import the files into a master blueprint <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Resource grouping: Allows resources to be grouped to ensure concurrent execution of their lifecycle operations <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Input number of instances </p>
    </li>

    <li>
      <p>Application blueprint versioning</p>
    </li>

    <li>
      <p>TOSCA composition and substitution</p>
    </li>

    <li>
      <p>TOSCA requirements and capabilities </p>
    </li>
  </ul>

  <h2>Command Line</h2>

  <ul>
    <li>
      <p>Simplify install process via CLI <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Mac Package of Cloudify CLI</p>
    </li>

    <li>
      <p>Ubuntu Package of Cloudify CLI</p>
    </li>
  
    <li>
      <p>Unify Cloudify Manager control via CLI and Cloudify CLI application execution with same experience</p>
    </li>
  </ul>

  <h2>Web UI Management Console </h2>

  <ul>
    <li>
      <p>Plugins CRUD Management in user interface <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Snapshots management in user interface <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Modernize Web Management user interface</p>
    </li>
  </ul>

  <h2>Provider Plugins</h2>

  <ul>
    <li>
      <p>Google Cloud Platform plugin <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Azure Plugin <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Kubernetes Plugin <strong>*In Progress*</strong></p>
    </li>

    <li>
      <p>Mesos Plugin</p>
    </li>

    <li>
      <p>Plugin extension mechanism </p>
    </li>

    <li>
      <p>Workflow extension mechanism </p>
    </li>

    <li>
      <p>Allow resume operation for failed workflows </p>
    </li>

    <li>Scheduled workflow execution engine</li>
  </ul></p>
