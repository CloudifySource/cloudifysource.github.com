Kubernetes Orchestration as a Service on OpenStack, AWS and Other Clouds

About The Service

Cloudify and Mist.io are proud to offer Kubernetes Orchestration-as-a-Service for quick, hassle free provisioning and configuring of a Kubernetes cluster. The orchestration service simplifies the provisioning of complex infrastructure and applications across heterogeneous environments.

We make setting up a Kubernetes cluster on AWS or OpenStack simple. With just a single click, a cluster can be set up in minutes, making it easy for users to get started with Kubernetes. New providers will be supported in the future.

Key Features

Easy Setup:
  • Create Kubernetes cluster in few min vs couple of hours
Auto-Scaling of Kubernetes Infrastructure
  • Expand or shrink the Kubernetes cluster (Minions) on demand
Extended Micro Services management
  • Auto scaling of microservices
  • Microservices composition
  • Management and monitoring of microservices
Works on Public and Private clouds
  • Run kubernetes on Public/Private cloud
  • Single pane glass
  • Optimize the cost across clouds

How It Works

The cloud orchestration service, based on Cloudify, the open-source, TOSCA-based orchestration platform enables you to create a single, reusable blueprint to provision the same application across multiple cloud providers, data centers, or hybrid environments. We've created a pre-baked Kubernetes blueprint that will allow you to setup a kubernetes cluster easily.

Rule based auto-scaling tools make it easy to scale up and down the cluster when necessary.

The service combines cloud orchestration with Mist.io features such as monitoring, scheduling, RBAC, and/or APIs, to create automated and dynamic workflows and policies that respond to system events in real time.

Keeping the Kubernetes-Native Experience

Kubernetes users will be able to use the exact same Kubernetes microservices and tools that they are used to with this new service.

With this new service we were primarily focused on taking the hassle out of your Kubernetes experience without shielding you from using the Kubernetes infrastructure directly as you would otherwise.
