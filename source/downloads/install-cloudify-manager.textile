<h2 class="name"><strong>Installing Cloudify Manager Via Packages</strong></h2>
<p>This method of installing Cloudify Manager is for users that have installed Cloudify CLI using one of our packages. Users who have installed Cloudify CLI via the script, follow the instructions further down the page.</p>

<p>Step 1: Ensure that you have properly installed <a href="http://docs.getcloudify.org/3.3.1/installation/from-packages/">Cloudify CLI</a>.</p>
<p>Step 2: Run <code>cfy init</code> in your Cloudify terminal. You should see the following:</p>
<pre class="sh language-sh" data-lang="sh"> <code class="sh hljs bash"> 
$ cfy init
Initialization completed successfully

...
 </code> </pre>
<p>Step 3: Edit the inputs.yaml file for your cloud provider located in the "/cloudify-manager-blueprints" folder of your installation with the required inputs.</p>
<p>Step 4: Run the bootstrap (install) command for your cloud environment as follows:</p>
<pre class="sh language-sh" data-lang="sh"> <code class="sh hljs bash"> 
$ cfy bootstrap --install-plugins -p /path/to/manager/blueprint/file -i /path/to/inputs/yaml/file
...

 </code> </pre>

 <p>Wait between 10-20 minutes for configuration and you should see the following when complete:</p>
 <pre class="bash language-bash" data-lang="bash"> <code class="bash hljs"> 
...

bootstrapping complete
management server is up at &lt;YOUR MANAGER IP ADDRESS&gt;

...
 </code> </pre>
<p>Step 5: Validate your installation by pointing your web browser to &lt;YOUR MANAGER IP ADDRESS&gt;:80 and you should see Cloudifyâ€™s Web UI. You now have an operational Cloudify Manager on the desired cloud provider!</p>

Now, go back to your terminal and run <code>cfy status<code> to verify the manager is running correctly and you should see something like this:</p>
<pre class="sh language-sh" data-lang="sh"> <code class="sh hljs bash"> 
$ cfy status
...

Getting management services status... [ip=46.137.95.124]

Services:
+--------------------------------+---------+
|            service             |  status |
+--------------------------------+---------+
| InfluxDB                       | running |
| Celery Management              | running |
| Logstash                       | running |
| RabbitMQ                       | running |
| AMQP InfluxDB                  | running |
| Manager Rest-Service           | running |
| Cloudify UI                    | running |
| Webserver                      | running |
| Riemann                        | running |
| Elasticsearch                  | running |
+--------------------------------+---------+

...
 </code> </pre>

 <p>Now you are ready to <a href="http://docs.getcloudify.org/3.3.1/manager/upload-blueprint/">upload a blueprint</a> for your Manager to use to deploy your application. Learn more about <a href="http://docs.getcloudify.org/3.3.0/blueprints/overview/">blueprints here</a>.</p>

<h2 class="name"><strong>Installing Cloudify Manager Via Python Script</strong></h2>
<p>This method of installing Cloudify Manager is for users that have installed Cloudify CLI using the script only. If you installed using one of our packages, please see the above instructions.</p>

<p>Step 1: Ensure that you have properly installed <a href="http://docs.getcloudify.org/3.3.1/installation/from-script/">Cloudify CLI</a>.</p>
<p>Step 2: Clone the "cloudify-manager-blueprints" repository from GitHub as follows:</p>
<pre class="bash language-bash" data-lang="bash"> <code class="bash hljs"> 
$ mkdir -p ~/cloudify-manager
$ <span class="hljs-built_in">cd</span> ~/cloudify-manager
$ git <span class="hljs-built_in">clone</span> https://github.com/cloudify-cosmo/cloudify-manager-blueprints
$ <span class="hljs-built_in">cd</span> cloudify-manager-blueprints
$ git checkout -b cloudify &lt;tag&gt;
...

 </code> </pre>
 <div class="bs-callout bs-callout-warning"><strong><p>Note </p></strong>
    <p>Make sure you use a &lt;tag&gt; from <a href="https://github.com/cloudify-cosmo/cloudify-manager-blueprints/releases">cloudify-manager-blueprints</a> that matches your Cloudify version. Blueprints taken from the master branch might not work for you.</p>

</div>

<p>Step 3-6: Same as above steps 2-5.</p>

<p>Now you are ready to <a href="http://docs.getcloudify.org/3.3.1/manager/upload-blueprint/">upload a blueprint</a> for your Manager to use to deploy your application. Learn more about <a href="http://docs.getcloudify.org/3.3.0/blueprints/overview/">blueprints here</a>.</p>
